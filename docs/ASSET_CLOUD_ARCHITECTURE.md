# 轻设云端素材库架构

## 当前实际部署（2026-07-11）

- 编辑器、项目文件和自动保存继续完全在本地运行。
- 素材运维页继续在本地完成抠图、裁边、缩略图和人工确认；服务器不运行 AI 图片处理。
- 已确认的成品通过“同步云端”上传，服务器只负责目录、编号、文件存储和编辑器下载。
- 当前访问地址为 `http://191.223.220.201/qingshe-assets/api/v1`，暂不配置域名与证书。
- 当前服务器约 1 核 CPU、1 GB 内存，因此批量选择仍然支持，但上传队列按单路顺序执行，避免并发大文件导致连接中断。
- 服务使用 Docker Compose、FastAPI、Caddy、SQLite 和服务器本地磁盘，没有新增付费组件。
- 原喜多多站点服务已停止并保留备份，没有删除原站点文件。
- Admin Token 只允许用于本机素材运维构建，不能放入将来交付给普通用户的编辑器安装包。

当前数据流：

```text
本机素材运维页
  -> 本地抠图/裁边/分类/确认
  -> 单路上传成品
  -> 服务器素材目录与文件
  -> 编辑器按需读取缩略图和透明 PNG
```

## 1. 结论

轻设采用“项目本地、素材云端、使用后缓存”的混合架构。

- Windows、macOS、iPad 编辑器继续在本地完成画布编辑和项目保存。
- 服务器只负责素材入库、处理、审核、检索、下载、回收站和备份。
- 使用过的素材必须保存在项目本地，服务器离线或素材下架不能破坏旧项目。
- 保留现有素材服务的编号、SHA-256 查重、SQLite、处理队列、透明抠图、缩略图、审核状态、版本表、分页和事件通知。
- 不建设账号系统、云项目、多人协作、微服务、Kubernetes、Redis、Kafka、对象存储或 CDN。

## 2. 系统边界

```text
素材管理后台
    │ 上传、审核、分类、删除、恢复
    ▼
HTTPS / 素材 API
    ├── SQLite：编号、分类、状态、版本、标签、任务
    ├── 服务器磁盘：原图、透明成品、缩略图
    ├── 单处理队列：查重、抠图、裁边、缩略图、自动分类
    └── 定时备份
    ▲
    │ 搜索、分页、缩略图、按需下载
Windows / macOS / iPad 编辑器
    ├── IndexedDB：已下载素材缓存
    └── 本地项目：图层参数和所用素材副本
```

### 服务器负责

- 素材的唯一编号和版本。
- 素材文件与元数据的统一管理。
- 入库处理、审核、搜索、分页和回收站。
- 只读下载接口与管理员写入接口。
- 数据库及素材文件备份。

### 编辑器负责

- 画布、图层、透视、缩放、导出等编辑行为。
- 本地项目自动保存和 `.qingshe` 项目备份。
- 素材缩略图与透明原图缓存。
- 服务器不可用时继续打开和编辑已有项目。

## 3. 复用的成熟组件

| 能力 | 采用方案 | 原因 |
|---|---|---|
| 后台业务 | 现有 Python `Catalog`、`AssetWorker`、分类器和抠图管线 | 已经可用，不重写 |
| HTTP API | FastAPI + Uvicorn | 复用成熟的请求解析、文件上传、鉴权和接口文档 |
| 元数据 | 现有 SQLite，保持 WAL 模式 | 单服务器、单管理员和上万素材足够 |
| 图片存储 | 服务器本地目录 | 简单、免费、可直接备份 |
| HTTPS 和反向代理 | Caddy | 自动证书、配置少 |
| 客户端请求 | 现有 `ky` + Zod | 已经接入，保留超时和边界解析 |
| 客户端缓存 | 浏览器 HTTP 缓存 + Dexie/IndexedDB | 已经属于当前技术栈 |
| 部署 | Docker Compose | 一个应用容器、一个 Caddy 容器即可 |
| 备份 | SQLite 在线备份 + restic 或服务器现有备份任务 | 不自研备份系统 |

现有 `ThreadingHTTPServer` 只用于本机开发。云端不直接暴露它，但它内部的目录、数据库和处理逻辑继续复用。

## 4. 数据目录

服务器只挂载一个持久化目录：

```text
/srv/qingshe-assets/
├── catalog.db
├── originals/
├── processed/
├── thumbnails/
├── models/
├── backups/
└── temporary/
```

- 原图、透明成品和缩略图不存进数据库。
- 数据库只保存相对路径，不能保存某台电脑的绝对路径。
- 上传先进入 `temporary`，处理成功后再原子移动到正式目录。

## 5. 素材身份与版本

每个素材同时保留三个标识：

- `id`：内部 UUID，永不变化。
- `code`：人可读编号，例如 `QS-000123`，永不复用。
- `version`：从 1 开始递增。

规则：

- 同一文件通过 SHA-256 查重，重复上传返回已有素材，不生成重复记录。
- 修改名称、分类和标签不增加文件版本。
- 替换透明成品或原图必须增加 `version`，禁止静默覆盖。
- 项目记录 `assetId + version`，同时保存实际使用的素材副本。
- 图片地址包含版本，例如 `/api/v1/assets/{id}/processed?version=2`，避免客户端拿到旧缓存。

## 6. 权限模型

不建设用户账号，只使用两类可轮换密钥：

| 密钥 | 能力 | 使用位置 |
|---|---|---|
| Editor Token | 查看、搜索和下载正式素材 | Windows、macOS、iPad 编辑器 |
| Admin Token | 上传、审核、修改、删除、恢复、备份 | 素材管理后台 |

- 请求统一使用 `Authorization: Bearer <token>`。
- 密钥只保存在服务器环境变量和客户端安全配置中，不写入 Git。
- 管理接口永远不接受 Editor Token。
- 所有外部访问必须经过 HTTPS。
- CORS 只允许正式管理后台地址；Tauri 客户端优先通过原生 HTTP 能力访问。

## 7. API 设计

所有接口放在 `/api/v1` 下，避免未来升级破坏旧客户端。

### 编辑器只读接口

| 方法 | 路径 | 用途 |
|---|---|---|
| `GET` | `/health` | 服务和磁盘状态 |
| `GET` | `/catalog` | 搜索、分类、审核状态和分页 |
| `GET` | `/catalog/revision` | 判断目录是否变化 |
| `GET` | `/assets/{id}/thumbnail` | 获取缩略图 |
| `GET` | `/assets/{id}/processed` | 获取透明成品 |

目录请求继续使用当前每页 120 项的策略。第一阶段保留 `limit + offset`，不一次同步整个素材库。

### 管理接口

| 方法 | 路径 | 用途 |
|---|---|---|
| `POST` | `/admin/assets/import` | 上传单张素材，批量由客户端并发 3 个请求 |
| `PATCH` | `/admin/assets/{id}` | 修改名称、分类、标签、收藏和审核状态 |
| `POST` | `/admin/assets/{id}/versions` | 上传新文件版本 |
| `DELETE` | `/admin/assets/{id}` | 移入回收站 |
| `POST` | `/admin/assets/{id}/restore` | 从回收站恢复 |
| `GET` | `/admin/jobs` | 查看处理任务 |
| `POST` | `/admin/jobs/{id}/retry` | 重试失败任务 |
| `POST` | `/admin/maintenance/backup` | 立即创建数据库备份 |
| `POST` | `/admin/maintenance/repair` | 数据库完整性和索引修复 |

### 事件通知

保留现有 SSE 事件，不增加 WebSocket：

- `asset.ready`
- `asset.updated`
- `asset.deleted`

iPad 从后台恢复时不能依赖长连接，因此编辑器同时在应用回到前台时检查一次 `/catalog/revision`。

## 8. 入库流程

```text
管理员选择图片
  → 客户端最多并发上传 3 张
  → 服务端检查格式和 25 MB 上限
  → SHA-256 查重
  → 分配 UUID 和 QS 编号
  → 写入任务队列
  → CPU 抠图
  → 根据透明像素裁紧边界
  → 生成 WebP 缩略图
  → 本地模型建议分类
  → 标记“待审核”
  → 人工确认
  → 出现在编辑器素材库
```

- 单张失败不能中断整批上传。
- 服务重启后，`processing` 任务恢复为 `pending` 并继续处理。
- 第一阶段不实现分片上传和断点续传；单文件上限只有 25 MB，没有必要增加复杂度。

## 9. 编辑器读取流程

```text
打开素材面板
  → 请求 120 条元数据
  → 浏览器加载小缩略图
  → 用户选择素材
  → 检查本地 assetId@version 缓存
  → 命中：直接使用
  → 未命中：下载透明成品并写入 IndexedDB
  → 加入画布
  → 项目保存素材副本
```

- 缩略图和透明成品响应使用 `Cache-Control` 与 `ETag`。
- 版本不变时客户端复用缓存；目录刷新不重复下载图片。
- 搜索和分类请求只返回元数据，不传输原图。
- 服务器离线时显示上次缓存的目录，已有项目继续编辑。

## 10. 删除与旧项目保护

- 删除默认是软删除，只写入 `deleted_at`。
- 被删除素材立即从新目录查询中消失。
- 已打开项目继续使用项目内的素材副本。
- 回收站默认保留 30 天，但不自动永久清除；永久清理由管理员明确执行。
- 永久清除前必须确认该素材已经存在于服务器备份中。

## 11. 缓存和同步

- 目录每次发生审核、更新、删除或恢复时增加一个单调递增的 `catalog_revision`。
- 编辑器保存最后一次 revision；相同则不刷新目录。
- 目录响应使用 `ETag`，无变化返回 `304 Not Modified`。
- 图片文件按 `assetId + version` 视为不可变，可长期缓存。
- 不同步云端项目，不同步画布状态，不做多人实时同步。

## 12. 部署结构

```text
Internet / 私有网络
        │ HTTPS 443
        ▼
      Caddy
        │ 127.0.0.1:7000
        ▼
FastAPI + AssetWorker
        │
        └── /srv/qingshe-assets 持久卷
```

第一阶段只运行一个应用实例。SQLite 和单处理队列不做横向扩容。

建议起步资源：

- 4 核 CPU。
- 8 GB 内存。
- 素材预计容量两倍以上的磁盘空间。
- 每天一次数据库和文件增量备份。

资源不足时，上传仍可排队，只是抠图速度变慢，不影响编辑器读取已完成素材。

## 13. 故障处理

| 故障 | 行为 |
|---|---|
| 服务器暂时离线 | 已有项目和已缓存素材继续使用 |
| 上传中断 | 该文件失败，其他文件继续；用户可重传 |
| 抠图失败 | 进入失败任务列表，可单独重试 |
| 重复上传 | 返回已有素材编号 |
| 数据库重启 | WAL 恢复，未完成任务重新排队 |
| 磁盘不足 | 拒绝新上传，保留读取能力并显示告警 |
| 素材被删除 | 新目录不可见，旧项目不缺图 |
| 客户端缓存损坏 | 按 assetId 和 version 重新下载 |

## 14. 实施顺序

1. 将硬编码的 `http://127.0.0.1:7000` 改为统一配置。
2. 为现有服务增加 `/api/v1` 路径和 Editor/Admin Token。
3. 用 FastAPI/Uvicorn 替换仅用于本机的 HTTP 入口，复用 Catalog 和 AssetWorker。
4. 将数据库文件路径改为相对持久卷路径。
5. 增加 `catalog_revision`、ETag 和按版本缓存。
6. 加入 Caddy、Docker Compose、健康检查和备份任务。
7. 将现有本地素材批量迁移到服务器并核对编号、文件数量和审核状态。
8. Windows 编辑器切换到云端素材地址。
9. 完成 iPad 触控适配后，复用同一 API。

## 15. 验收标准

- 素材库拥有 10,000 条记录时，客户端仍只加载当前 120 项。
- 批量上传时并发数不超过 3，单张失败不影响其他文件。
- 相同文件重复上传不会产生新编号。
- 未审核素材不会出现在编辑器。
- 删除素材后目录立即消失，旧项目仍可打开且不缺图。
- 服务器断开时已有项目可以继续编辑和导出。
- Editor Token 无法调用任何写入接口。
- 服务重启后待处理任务能够继续。
- 能从备份恢复数据库、原图、透明成品和缩略图。

## 16. 明确不做

- 不做用户注册、登录、组织和角色管理界面。
- 不做云端项目和多人协作。
- 不做素材商城和付费系统。
- 不接入付费图片处理或生成 API。
- 不使用微服务、消息中间件、Kubernetes、Redis、PostgreSQL、S3 或 CDN。
- 不为 Windows、macOS 和 iPad 各自开发不同的素材协议。
