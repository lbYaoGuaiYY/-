# 轻设全自动素材流水线设计

## 目标

用户在 Chrome/Edge 或 Firefox 的轻设插件中选择 ChatGPT/Gemini、填写提示词和数量后点击一次“开始”；插件完成新对话、逐张生成、发现、去重、上传和建立抠图任务，在线抠图器完成处理后，轻设 App 无需刷新就出现成品。

## 产品边界

- “当前页面”手动模式保留现有“发送到素材面板后等用户确认”的门禁。
- “全自动”模式的“开始”按钮是一次明确授权；运行内可直接上传并建任务，不再逐张确认。
- 首批生成器只包含 ChatGPT 和 Gemini。AI Studio、Claude 和 Copilot 继续支持手动扫描，不假装具备稳定的全自动适配。
- 数量为 10 时，使用同一个新对话串行发出 10 次“生成 1 张”请求，避免依赖生成站点对“一次返回 10 张”的不稳定解释。
- 全自动默认串行，避免同账号并发对话导致限流或抓错图。

## 架构

### 浏览器插件

- Manifest V3 同时声明 `background.service_worker` 和 `background.scripts`；Chrome 121+ 选用 worker，Firefox 121+ 选用 scripts。
- 内容脚本用 `MutationObserver` 长驻观察 DOM，发现新的有效大图后发送摘要事件，后台更新角标和页面收件箱。
- ChatGPT/Gemini 适配器只负责新对话 URL、编辑器定位、提交和生成图识别；编排状态由后台脚本持久化到 `storage.local`。
- 后台脚本不保存大图到浏览器存储。图像发现后立即以扩展设备令牌上传服务器。
- 每个项目使用 `run_id + item_id` 作幂等键，worker 重启或页面重复消息不会建立重复任务。

### 服务器

- 新增 `extension_devices`、`automation_runs`和 `automation_items` 持久化表。
- 素材面板登录会话只用于配对扩展设备；发给扩展的令牌只能心跳、管理本设备的自动运行和上传自动运行图片，不具备素材库管理权限。
- 项目状态：`queued -> generating -> uploading -> processing -> ready`，失败为 `failed`，可在扩展中重试当前项。
- 扩展上传端点验证设备、运行和项目归属，校验 MIME/体积，然后直接复用 `RemoteProcessingStore.create_task()`。
- 处理节点每次 poll 都刷新 `last_seen`；扩展每 30 秒心跳。素材面板的同一个 dashboard 返回两类设备和运行进度。

### 素材面板与轻设 App

- 素材面板显示扩展设备、处理节点在线状态、当前自动运行和每项进度。
- 配对页通过现有页面内容脚本桥把限权扩展令牌交给后台脚本。
- 轻设 App 在可见时每 5 秒读取低成本的 `catalog/revision`；修订号变化时才重载首页素材。

### 轻设抠图器

- 用 Tauri 2 窗口替换 Tkinter 界面，现有 Python/rembg 处理核心作为打包 sidecar，不改服务器任务协议。
- UI 遵守 `DESIGN.md`：`#161616/#252525/#303030`表面、1px 边线、4px 圆角、无渐变/阴影/玻璃、蓝色只用于主操作和焦点。
- 窗口显示服务器连接、节点身份、当前任务、本次已完成数、最近状态和日志入口。关闭窗口最小化，“退出”才停止 sidecar。
- sidecar stdout 每行输出 JSON 事件，Tauri 前端不解析人类日志。

## 故障恢复

- 扩展 worker 被回收：活动运行保存在 `storage.local`，内容脚本的发现消息会重新唤醒 worker。
- 生成页被关闭：项目标记失败，弹窗提供“继续”，不悄悄新建无限页签。
- 选择器失效或站点限流：显示可见错误和当前项序号；每项最多自动重试 2 次。
- 服务器/抠图器离线：已上传任务保持 `pending`，节点恢复心跳后自动领取。
- 重复图：项目幂等键防止重复任务，素材库再用内容 SHA-256 做最终去重。

## 验收标准

1. Chrome 加载解压包，Firefox 通过本地临时安装加载同一代码产物；两者的弹窗、长驻扫描和后台脚本正常。
2. 生成页后续新增图片时，不打开弹窗也会更新插件角标。
3. 在插件输入“婚庆素材”、10 张，可见观察到新对话、自动输入、逐张上传、任务处理和 10 个成品出现。
4. 素材面板能区分在线/离线扩展和抠图器，显示运行的总数、完成数和失败项。
5. 轻设 App 保持前台时，新成品在不重启、不切窗的情况下出现。
6. 本地打包的抠图器双击显示现代化窗口，配对、心跳、领取和完成真实任务的链路通过。

## 发布限制

- Chrome/Edge 本地安装可用解压包完成。
- Firefox Release/Beta 的永久安装包需要 Mozilla 签名；代码库产出可签名 XPI，本机验收用 `web-ext run`临时安装，AMO 凭据不写入仓库。
- macOS 抠图器本机可用 ad-hoc 签名验收；对外分发的无提示安装仍需 Developer ID 签名和公证。
