# 轻设完整产品与素材云控制面设计

## 1. 目标

将当前已能运行的轻设编辑器、素材入库工具和云素材 API 整理为一套可交付产品：

- Windows、macOS、iPad 和网页端只通过 `https://xiduoduo.top/qingshe-assets/api/v1` 传输素材。
- `191.223.220.201` 只是源站层的备用信息，不写入编辑器源码、环境文件、安装包、运行日志或用户可见文案。
- 普通编辑器保持画布优先，只显示与当前设备相关的素材连接、下载和缓存状态。
- 独立素材管理端承载入库、审核、云同步、容量、客户端、传输、告警和安全控制。
- 整体视觉改为接近 Photoshop / Affinity 的中性炭黑，蓝色只用于焦点、选区和必要的主操作。

## 2. 当前根因与整理边界

当前编辑器的类型检查、100 个单元测试、静态检查、生产构建和 49 个端到端流程均通过，因此不做无目的架构重写。整理聚焦于已证实的交付缺口：

1. 云端 `/health` 只返回 `ready`，无法识别资源压力、容量和服务退化。
2. 云端没有客户端活跃度、下载字节、失败率、活跃传输和历史窗口统计。
3. 素材管理端同时承担本地处理和云同步，但缺少明确的云端运行视图。
4. 当前运行环境生成脚本会把 `191.223.220.201` 写进编辑器环境，与源站隐藏目标冲突。
5. Caddy 只监听 HTTP 80，而 `xiduoduo.top` 的 HTTPS 当前返回 Cloudflare 521，线上链路未达到可交付状态。
6. 现有色板虽然是深色，但控件、选中背景和强调色整体偏蓝。

不改动画布模型、项目本地存储、素材审核门禁、本地缓存和使用后随项目固化的已验证设计。

## 3. 产品分层

### 3.1 普通编辑器

编辑器不新增后台入口，遵守 `DESIGN.md` 的产品边界。素材面板只提供：

- `在线`、`连接较慢`、`离线，正在使用缓存` 三类用户可理解状态。
- 当前设备的活跃下载数、进度、失败与重试。
- 本地缓存的素材数量和估算占用。

不向普通用户显示 CPU、内存、磁盘、IP、客户端清单、维护开关或管理员密钥。

### 3.2 素材管理端

保留已有入库、抟图、裁边、分类、待检查、在库和回收站。顶部新增三个紧凑的运行视图：

- **概览**：连接状态、最后更新时间、运行时长、CPU、内存、磁盘、素材数、库文件大小。
- **客户端与传输**：最近 5 分钟活跃客户端数、当前下载数、最近 24 小时请求数、下载次数与字节数、失败数、近期活跃设备。
- **运维与风险**：维护模式、新下载开关、最大并发下载、备份时间、告警列表、刷新、备份、重试失败任务。

控制操作全部需要 Admin Token，显示确认语句，返回操作结果，并写入审计记录。首期不提供远程重启主机、任意 Shell 或删除备份，避免将 Web 管理页变成远程入侵入口。

## 4. 云端遥测与控制

### 4.1 轻量数据模型

云 API 进程内维护最近 24 小时的分钟级聚合数据，持久化到 `/data/telemetry.json`。不记录原始 IP；设备标识由客户端随机生成，服务端仅保留加盐哈希和粗粒度平台类型。

每次请求聚合：

- 时间窗口、路由类别、状态码类别、响应时长。
- 素材媒体下载次数和响应字节。
- 活跃请求数和媒体下载数。
- 设备哈希、平台、应用版本、最后活跃时间、当日请求和下载字节。

主机指标只从 Linux `/proc` 和 `statvfs` 读取，不引入高常驻内存依赖：

- CPU 利用率、负载均值。
- 内存已用/总量和利用率。
- 素材卷已用/总量/可用量和利用率。
- 服务运行时长。
- 素材总数、已审核、待检查、回收站、原图/成品/缩略图字节数。

### 4.2 API

- `GET /health`：匿名、极小响应，只返回 `ready | degraded | maintenance`、版本和服务时间，用于 CDN 健康检查。
- `GET /admin/observability/summary`：返回主机、库、活跃客户端、传输、控制和告警摘要。
- `GET /admin/observability/clients`：返回最近 24 小时的匿名设备活跃摘要。
- `GET /admin/observability/transfers`：返回最近 24 小时分钟级请求、下载字节和失败趋势。
- `PATCH /admin/controls`：仅允许修改 `maintenance_mode`、`downloads_enabled` 和 `max_concurrent_downloads`。

公开健康接口不暴露主机资源、库统计、客户端或版本细节。

### 4.3 告警与退化

默认阈值：

- 内存使用 `>= 85%` 警告，`>= 95%` 严重。
- 磁盘使用 `>= 80%` 警告，`>= 90%` 严重。
- 最近 5 分钟失败率 `>= 5%` 警告，但总请求少于 20 时不触发。
- 无法读取素材卷或 SQLite 完整性检查失败为严重。

维护模式下保留健康检查、管理端和已缓存项目；编辑器的新目录和媒体下载返回 `503` 与可重试信息。只关闭新下载时，目录仍可读，媒体返回 `503`。

## 5. 客户端身份与传输

编辑器首次启动生成随机 UUID，存入当地应用存储。每个 API 请求附带：

- `X-Qingshe-Client`：随机设备 ID。
- `X-Qingshe-Platform`：`windows | macos | ios | web`。
- `X-Qingshe-Version`：应用版本。

这些字段只用于运营统计，不是鉴权凭证。Editor Token 可被反编译取得，因此只允许读取已审核素材，并配合 CDN 限流；Admin Token 不得进入普通编辑器构建。

媒体 URL 不再在 query string 中放长期 Editor Token。为了与 `<img>` 和缓存兼容，服务端提供短时效 HMAC 签名媒体 URL，目录请求返回可使用的签名参数；缓存键仍以 `assetId + version` 为准。

## 6. 域名、TLS 与源站隐藏

交付架构：

```text
编辑器 / 素材管理端
  -> HTTPS xiduoduo.top
  -> Cloudflare 代理、健康检查和限流
  -> 源站 Caddy :443
  -> qingshe-assets:7000
```

- 所有生产配置仅允许 HTTPS 且 hostname 必须等于 `xiduoduo.top`。
- Caddy 同时提供 80 到 443 跳转和 443 源站服务。
- Cloudflare 保持代理，健康检查路径为 `/qingshe-assets/api/v1/health`。
- 源站防火墙的 80/443 只允许 Cloudflare 公布的 IP 段；SSH 只允许管理网络或现有安全入口。
- 仓库中清理生产 IP；部署脚本从本机私有环境变量读取源站，不生成到客户端文件。

无法对已历史暴露的 IP 做绝对保密承诺，但可以保证新客户端、DNS 响应和应用日志不再暴露它，并让直连源站无法获得素材服务。

## 8. 错误处理和安全

- 遥测写入失败不得中断素材读取，管理端显示“统计暂不完整”。
- 主机指标不可用时返回 `null + reason`，不伪造 0。
- 控制配置采用原子写入，值域由 Pydantic 验证，服务重启后恢复。
- 不记录 Token、完整签名 URL、原始 IP、文件内容或项目内容。
- 所有管理 API 继续使用 Admin Token，普通编辑器的 Editor Token 无法读取遥测或修改控制。
- 前端对网络失败、`401/403`、`429`、`503` 给出不同反馈，不将所有错误简化为“服务未启动”。

## 9. 验证与交付

### 自动验证

- Python API 测试覆盖权限、指标计算、客户端匿名聚合、传输计数、控制持久化、维护模式和告警。
- TypeScript 测试覆盖生产域名限制、客户端标识头、管理响应解析和错误分类。
- Playwright 覆盖管理概览、离线/退化、控制确认、编辑器连接状态与黑色主题。
- 运行 `pnpm typecheck`、`pnpm test`、`pnpm check`、`pnpm build`、`pnpm test:e2e`。
- 使用 Python 3.12+ 运行 `tests/test_asset_cloud.py`。

### 可见验收

- 实际启动编辑器和素材管理页，验收桌面宽度、平板竖屏和手机宽度。
- 验证画布、左右面板、素材预览、选中态、焦点和管理告警的对比度。
- 安装包和 `dist` 全文扫描，不得出现 `191.223.220.201`、Admin Token 或非 `xiduoduo.top` 的生产素材端点。

### 线上验收

- 从 Windows 管理通道 SSH 进入源站，部署并启动 Docker Compose。
- `https://xiduoduo.top/qingshe-assets/api/v1/health` 返回 200，证书有效。
- 管理端可见真实主机、库、客户端和传输数据。
- 普通 Editor Token 访问管理遥测和控制接口均被拒绝。
- 新构建的 Web、Windows、macOS 和 iPad 配置只包含 `xiduoduo.top`。

## 10. 实施顺序

1. 先用测试固定域名限制和云端遥测/控制合约。
2. 实现云端轻量遥测、统计中间件、控制持久化和管理 API。
3. 实现客户端标识、连接状态和管理端运行视图。
4. 修改 token 与组件样式，完成中性炭黑主题和三端可见验收。
5. 清理生产 IP、更新 Caddy/Compose/文档，扫描构建产物。
6. 从 Windows 进入源站部署、启动，完成 HTTPS、权限、监控数据和端到端传输验收。
