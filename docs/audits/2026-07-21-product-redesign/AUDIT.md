# 轻设产品体验与三端一致性审计

日期：2026-07-21

## 审计范围

- Windows/macOS 桌面编辑器共享体验
- iPadOS 横屏响应式体验
- 项目入口、素材浏览、画布编辑与图层操作
- 用户投稿、审核与抠图流程的现状边界
- 三端版本、构建、发布与更新链路

## 流程证据

1. **桌面编辑器总览 — 可用，但素材状态与主操作层级不清晰**
   - 截图：[01-editor-overview.png](01-editor-overview.png)
   - 素材、画布、属性和图层形成完整三栏工作区。
   - 云服务不可用时会回退本地缓存，但“本地模式/使用本地缓存/正在读取”同时出现，状态表达分散。
   - 素材卡片存在失效预览时仍占据主要空间，缺少统一错误态与恢复动作。

2. **项目列表 — 健康，但只强调本地保存**
   - 截图：[02-project-home.png](02-project-home.png)
   - 新建、打开、重命名、复制和删除入口清楚。
   - “仅保存在当前设备”与用户期待的三端同步相冲突；当前项目格式共享，但项目本身尚未云同步。
   - 项目卡片操作只用图标，需要更稳定的提示与触屏命中范围。

3. **iPad 横屏编辑器 — 画布优先正确，核心入口不足**
   - 截图：[03-ipad-landscape-editor.png](03-ipad-landscape-editor.png)
   - 画布获得主要空间，符合画布优先原则。
   - 1194px 宽度下已切换抽屉，而设计规范声明 1180px 以上应保持三栏，代码断点与规范不一致。
   - 顶部工具密度仍按桌面排列，触屏操作需要更明确的分组和 44px 命中区。

4. **iPad 素材抽屉 — 可浏览，但无法完成投稿闭环**
   - 截图：[04-ipad-asset-panel.png](04-ipad-asset-panel.png)
   - 抽屉和遮罩能保护画布上下文，搜索与分类可继续使用。
   - 当前只有刷新、离线管理和素材插入，没有“提交素材”和“我的提交”。
   - 抽屉关闭时未确认焦点返回；隐藏面板仍在 DOM 中，存在键盘焦点越界风险。

5. **1440px 桌面工作区 — 结构完整，信息反馈可收敛**
   - 截图：[05-desktop-editor-1440.png](05-desktop-editor-1440.png)
   - 三栏比例和画布中心性良好，应作为 Windows/macOS 共同视觉基线。
   - 属性空态、素材加载态和顶部状态占用了多个注意点，可整合为更少、更直接的工作状态。

## 最高优先级问题

1. **P0 — 投稿能力缺失**：App 只有只读 Editor Token；后台处理、审核和管理员上传链路已经存在，但不能安全地直接开放给 App。
2. **P0 — 三端发布不是单一来源**：核心前端同源，但版本、构建号、Xcode 生成工程、下载文件名和发布产物仍在多处手工维护。
3. **P1 — iPad 断点偏差**：设计规范与代码断点相差 100px，导致 1180–1279px 设备过早退化为抽屉布局。
4. **P1 — 投稿状态不可见**：处理队列只有后台视角，用户无法知道等待、抠图、待审核、通过或失败。
5. **P1 — 弹层与抽屉焦点管理不足**：弹层缺少焦点圈定、关闭后恢复，画布对象缺少辅助技术可读替代。
6. **P2 — 状态语言分散**：本地缓存、云服务、自动保存和素材加载各自发声，缺少统一的连接/同步状态模型。

## 产品决策

- 保持轻设为一个主产品，不复制 Windows、macOS、iPadOS 三套业务 UI。
- 继续使用同一套 React/Fabric 编辑器；平台差异只放在响应式组件和平台适配层。
- 新增受限的 `Submission` 能力，使用独立投稿凭据与状态凭据，绝不提升 Editor Token 或暴露 Admin Token。
- 默认投稿进入抠图并待审核；用户也可选择“保留原图，仅审核”。任何图片都不会直接进入正式素材库。
- App 增加“提交素材”和“我的提交”，用清晰状态展示等待、处理中、待审核、已通过和失败。
- 视觉继续遵守现有深色、边线分层、蓝色只用于主操作与选中态的设计系统。

## 证据边界

- 当前审计使用 Web 运行时验证共享 UI；未在本机实际运行 macOS 或 iPadOS 原生文件选择器和系统分享面板。
- 截图不能证明完整键盘焦点循环、屏幕阅读器画布导航或真实网络延迟，需要在实现后单独测试。
- 当前 Windows 环境不能构建或签名 macOS/iPadOS 安装包；最终需由 macOS CI 与 Apple 签名环境验证。
